APIdone

A ready to use API

Start creating your project right away, you only have to worry about the client side.

Use it

APIdone, doesn't have any structure, you create the structure that fits your needs.

TODO list example

Let's create a simple TODO list, using curl.

This will be the API

Method url description
GET /tasks List tasks
POST /tasks Create task
PUT /tasks/:id Update task
PUT /tasks/:id Create task with :id (if :id doesnt exists)
DELETE /tasks/:id Delete task
GET /tasks?status=done Lost done tasks
GET /tasks?status=pending List pending tasks
POST /tasks/:id/notes Create a task note
GET /tasks/:id/notes List task notes

When you start using APIdone, your API is already done, let's use it.

Create Task

Command

curl http://demo.apidone.com/tasks \
	-d '{"name": "Buy milk", "status": "pending"}' \
	-H "Content-Type: application/json"

Response

If no id is send in the POST data, will create an id for you 
{
	"id":"4f529689643aec0100000003"
}

Update Task

Command

curl -X PUT -d '{"name": "Buy milk", "status": "done"}' -H "Content-Type: application/json" http://demo.apidone.com/tasks/4f529689643aec0100000003

Response

[
	{
		"id":"4f528be517952b0100000002",
		"name":"List task"
	}
]

Create another task

Command

curl -d '{"name": "Write review", "status": "pending"}' -H "Content-Type: application/json" http://demo.apidone.com/tasks

Response

}
